AICMS 不是
另一套 CMS，而是 AI 搜尋時代的商務基建。

AICMS 不是要取代 WordPress。在「給 Google 看」的時代，WordPress 是合適的工具。但 AI 搜尋接管 30–50% 流量後，網站該做的事改變了 — 從「給人讀」進化為「被 AI 引用，且在對話中直接成立訂單」。AICMS 從架構層為此而生。

為什麼做 AICMS三件正在發生、但中小企業尚未準備好的事。

這三件事是 AICMS 所有功能的設計依據。不是技術展示，而是替客戶把基建準備好。

01

AI 搜尋接管流量

ChatGPT、Claude、Perplexity、Google AI Overview 直接給答案並附引用來源。沒被 AI 引用，在新世代等同不存在。傳統 SEO 工具無法觀測對話式查詢。

02

對話成單成為新常態

消費者已習慣以對話解決事情，「填表單、等回覆」的流程在 AI 時代屬於過時體驗。新一代客戶期待的是「一句話即可獲得報價並完成下單」。

03

LLM 成為新的商務通路

Booking.com、OpenTable、Shopify 已有自己的 MCP server 供 LLM 串接，客人在 ChatGPT 直接訂機票與飯店。中小企業缺乏工程資源建置這層基建。

與其他方案的差異不是「另一套架站工具」，而是世代之差。

同樣是把網站架起來，AICMS 與主流方案的根本差別在於「面向的對象是誰」。WordPress 設計給 Google 爬蟲；AICMS 設計給 LLM 引用與 agent 直接下單。

技術選型每個決策都為「AI 搜尋時代」這個前提服務。

不堆砌外掛、不堆疊服務，只選真正解決問題的工具。

FRONTEND

Next.js 15 App Router + ISR

結構化資料於 build 時即生成，AI 爬蟲與一般使用者皆取得完整 HTML。熱門頁採 ISR 快取，由 CDN 直送，成本與速度同時最佳化。

BACKEND

Supabase + Row Level Security

多租戶 SaaS 架構，每位客戶資料完全隔離。內建 Auth、Edge Functions、Storage，不重複造輪子。

AI

多模型可選（Claude / GPT / Gemini）

不綁定單一供應商。AI 客服、內容生成、schema 補強各自選擇最適模型，費用、品質、合規同時優化。

MCP

Model Context Protocol Gateway

每位客戶自動具備一台 MCP server，ChatGPT、Claude、Perplexity 可直接連線，使您的服務出現在 LLM 的可呼叫清單中。

DEPLOY

Zeabur + Cloudflare DNS

主機部署於亞洲節點，台灣使用者延遲 <50ms。Cloudflare DNS only（不走 proxy），由我們自行掌控 SSL 與 cache header。

CMS

TipTap 編輯器 + Schema-first 內容模型

非 page builder 架構。內容與 schema 結構綁定 — 寫一篇文章自動產 Article schema、寫一條 FAQ 自動產 FAQPage schema。

實戰驗證我們自己使用，客戶才敢使用。

AICMS 每個核心功能皆先在自家或合作站做 dogfood，驗證實際可用、能真正轉換訂單後，才開放給客戶。
